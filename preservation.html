Monastery Preservation System

AI-Powered Image Comparison for Cultural Heritage Monitoring

üèõÔ∏è About the Preservation System

The Monastery Preservation System is an advanced AI-powered tool for monitoring structural changes in monastery buildings. It uses computer vision and structural similarity analysis to detect and track deterioration over time.

Key Features:

  • MongoDB GridFS Storage - Secure image storage
  • AI-Powered Comparison - OpenCV and SSIM analysis
  • Real-time Processing - Python microservice for image analysis
  • Visual Difference Highlighting - Annotated images showing changes
  • Severity Classification - Automatic assessment (EXCELLENT, GOOD, MODERATE, POOR, CRITICAL)
  • Comprehensive Reports - Downloadable analysis reports
  • Responsive UI - Modern React interface

üöÄ Quick Start

The preservation system requires a local setup with MongoDB, Node.js backend, and Python service. Follow these steps to run it:

Prerequisites:

  • Node.js 18.0+
  • Python 3.8+
  • MongoDB 5.0+

Installation Steps:

# 1. Navigate to preservation folder
cd Monastery-Preservation

# 2. Setup Backend
cd backend
npm install
# Create .env file with MongoDB URI

# 3. Setup Python Service
cd ../python-service
python -m venv venv
.\venv\Scripts\Activate
pip install -r requirements.txt

# 4. Setup Frontend
cd ../frontend
npm install

Running the System:

Open 3 separate terminals:

# Terminal 1: Backend (port 5000)
cd Monastery-Preservation/backend
npm run dev

# Terminal 2: Python Service (port 5001)
cd Monastery-Preservation/python-service
.\venv\Scripts\Activate
python app.py

# Terminal 3: Frontend (port 5173)
cd Monastery-Preservation/frontend
npm run dev
Open Preservation System (Local) View Full Documentation
